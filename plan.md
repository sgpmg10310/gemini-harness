# 프로젝트 계획서 (plan.md): Nh Ninja V4.3 (에셋 독립형 절차적 생성 및 시스템 안정화)

## 1. 프로젝트 개요 및 배경
- **목적:** 외부 이미지 URL 종속성을 완전히 제거하여 네트워크 환경에 무관한 안정적인 게임 실행 환경 구축.
- **핵심 변화:** 외부 리소스 로딩 방식에서 Phaser Graphics API를 이용한 **'캐릭터 절차적 생성(Procedural Generation)'** 방식으로 전환.
- **개선 목표 (V4.3):**
  - **에셋 독립성 확보:** 모든 캐릭터 에셋을 런타임에 코드로 생성하여 403/404/CORS 에러 가능성 원천 차단.
  - **시각적 정체성 유지:** 나루토, 사스케, 사쿠라, 카카시의 특징을 기하학적 도형과 색상 데이터로 재현.
  - **런타임 효율성:** 생성된 그래픽을 텍스처로 캐싱하여 렌더링 성능 최적화.
  - **물리 무결성:** 캐릭터 사망 및 게임 종료 시 물리 엔진 인터럽트를 확실히 처리하여 논리적 오류 방지.

## 2. 기술 스택 및 아키텍처 (V4.3)
- **게임 엔진:** Phaser 3 (JavaScript ES6+)
- **그래픽 시스템:** Phaser.GameObjects.Graphics (Texture Generation)
- **데이터 구조:** 캐릭터별 시각적 특징(머리카락, 의상, 눈 등)을 정의하는 JSON 형태의 상수 데이터.
- **물리 엔진:** Phaser.Physics.Arcade

## 3. 주요 구현 계획
- **캐릭터 생성기 (Character Generator):** Graphics API를 사용하여 원, 사각형, 다각형 등을 조합하여 캐릭터 드로잉.
- **텍스처 캐싱 워크플로우:** `PreloadScene`에서 런타임 생성 -> `generateTexture` 호출 -> 전역 텍스처 매니저 등록.
- **사망 로직 강화:** `kill()` 메서드 호출 시 `body.enable = false` 및 `setVelocity(0)`를 즉시 실행하여 물리 연산 중단.

## 4. QA 및 무결함 원칙
- 외부 리소스 없이도 모든 캐릭터가 식별 가능한 형태로 렌더링되는지 확인.
- 게임 종료 시점에 모든 물리 연동 객체의 움직임이 즉시 중단되는지 검증.
