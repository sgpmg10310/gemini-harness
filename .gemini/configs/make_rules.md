# [설정] 무결점 구현 단계 (/nh:make)

## 1. 단계 목표
- 설계에 기반하여 실제 작동하는 코드를 작성하며, 디자인 가이드 및 외부 에셋을 통합합니다.

## 2. 서브 에이전트 지침 (Make Agent)
- **로직 기억 및 보존:** 에이전트는 현재까지 구현된 소스 코드의 핵심 로직과 구조를 완벽히 기억해야 합니다.
- **최소 수정 원칙:** 사용자가 명시적으로 수정을 요청한 부분에 한해서만 코드를 변경합니다. 불필요한 전체 리팩토링은 금지됩니다.
- **대화형 인터랙션:** 수정을 진행하기 전, 변경할 범위와 로직을 사용자에게 대화형으로 물어보고 확인을 받은 뒤 실행합니다.
- **수정 제한:** 기존에 안정화된 기능을 파괴할 가능성이 있는 대규모 수정은 반드시 위험성을 고지하고 승인을 얻어야 합니다.

## 3. 검증 기준
- 코드가 설계서(`add.md`)를 100% 준수하며 작성되었는가?
- 클린 코드 원칙을 준수하고 중복을 최소화했는가?
- 사용자의 의도와 다른 임의의 코드 삭제가 없는가?

## 4. 디자인/에셋 연동 (MCP)
- 지정된 외부 리소스에서 요소를 분석하여 코드에 반영하고 결과를 보고함.

## 5. 필수 포함 항목
- 구현된 파일 목록 및 핵심 로직 설명
- 이전 로직 대비 변경된 부분에 대한 구체적 명시
- 구현 중 발생한 기술적 이슈 및 해결 방법 요약
