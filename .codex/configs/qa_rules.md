# [설정] 품질 감사 및 보안 검토 단계 (/nh:qa)


## 0. 필수 사항
- 최초의 질문은 "너 정말 이거 작동 되는거 맞아?" 가 핵심입니다.

## 1. 단계 목표
- 구현된 제품의 기술적 완결성, 보안성, 안정성 및 **시각적 무결성**을 전수 조사합니다.
- 실제 화면 하나하나를 스크린샷을 찍어 검토하는 수준의 정밀한 시각적 검증을 수행합니다.

## 2. 서브 에이전트 지침
- **역할:** SDET, 시각적 품질 감독(Visual QA), 및 코드 품질 감사관.
- **검증 기준 (Critical):**
  - **시각적 스크린샷 검증:** 다음 각 화면(State)에 대해 스크린샷을 분석하듯 시각적 요소를 개별 검토함.
    1. **Preload/Title:** 로딩 바의 정확도, 타이틀 텍스트의 정렬 및 효과.
    2. **Selection:** 캐릭터 이미지의 선명도, 선택 카드 레이아웃, 마우스 오버 반응.
    3. **In-Game:** 캐릭터/적의 시인성, 배경 패럴랙스 끊김 여부, UI(HP/Chakra) 가독성.
  - **코드 무결성:** JavaScript 문법 에러, 런타임 예외 가능성, 씬 전환 로직 안정성.
  - **린트 및 품질:** 변수 명명법, 일관된 스타일, 세미콜론, 들여쓰기 준수 여부.
- 스펙 주도 개발 방법론에 개발된 프로젝트가 맞는지 질문하세요. 맞다면 테스트 수행, 아니면 스펙 주도 개발 진행 요청
- TDD 방식에 의해 최소 단위(method, function)으로 테스트 수행
- 테스트 방식은 완벽도 100% 도달이며, 에러 메세지 하나 없는 정상 기동이 목표입니다.


## 3. 스코어링 및 피드백 루프
- **최종 점수:** 0.0 ~ 1.0 (최소 0.95점 이상 'Go')
- **무결점 원칙:** 시각적 배치가 어긋나거나, 이미지 로딩 실패, 혹은 단 하나의 코드 에러라도 발견될 시 즉시 No-Go 판정.
- **자동 수정 지시:** 에이전트는 발견된 오류(시각적/기술적)에 대해 즉시 수정 방안을 제시하거나 직접 수정을 시도해야 함.

## 4. 필수 포함 항목
- **화면별 시각적 검증 보고서 (스크린샷 분석 결과 포함)**
- 정적 분석 리포트 (문법, 린트, 품질 지표)
- 기능 및 보안 테스트 결과 목록
- 최종 시스템 구성도 (system_map.md) 생성
- 최종 승인 여부 (Go/No-Go)
